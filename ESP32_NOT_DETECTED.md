# ESP32 не определяется Windows - Решение

## Проблема
Windows вообще не видит ESP32 - нет COM-портов, устройство не определяется.

## Причины (по частоте)

### 1. Кабель только для зарядки (90% случаев!)
**Самая частая проблема!** Многие USB кабели поддерживают только зарядку, но не передачу данных.

**Решение:**
- Используй кабель, который точно поддерживает передачу данных
- Обычно это кабели от телефонов/планшетов, которые работают с компьютером
- Кабели от дешевых зарядок часто только для питания

### 2. Нужно нажать кнопку BOOT
Некоторые ESP32 требуют перехода в режим загрузки.

**Решение:**
1. Нажми и **удерживай** кнопку **BOOT** (или **IO0**)
2. Нажми и отпусти кнопку **RESET** (или **EN**)
3. Отпусти кнопку **BOOT**
4. Устройство должно определиться

### 3. USB порт не работает
Попробуй другой USB порт (желательно USB 2.0, не USB 3.0).

### 4. Плата неисправна
Проверь:
- Горят ли светодиоды на плате?
- Есть ли питание?
- Не перегревается ли плата?

## Проверка

Запусти скрипт для проверки:
```powershell
.\full_check.ps1
```

Или вручную:
```powershell
# Проверка COM-портов
[System.IO.Ports.SerialPort]::GetPortNames()

# Проверка устройств
Get-PnpDevice | Where-Object { $_.InstanceId -like "*VID_303A*" }
```

## После подключения

Когда устройство определится, но драйверы не установлены:

1. **Для ESP32-S3/C3 (нативный USB, VID_303A):**
   - Скачай: https://github.com/espressif/usb-serial-esp32/releases
   - Или установи через PlatformIO: `pio platform install espressif32`

2. **Для CH340 адаптера:**
   - Скачай: http://www.wch.cn/downloads/CH341SER_EXE.html

3. **Для CP210x адаптера:**
   - Скачай: https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers

## Автоматическая проверка

Запусти скрипт `watch_esp32.ps1` - он будет проверять подключение каждые 2 секунды:
```powershell
.\watch_esp32.ps1
```
